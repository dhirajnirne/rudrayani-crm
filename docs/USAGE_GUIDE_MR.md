(व्यवस्थापनासाठी) आणि मोबाइल अ‍ॅप (फील्ड/कॉलिंग कर्मचाऱ्यांसाठी).*

> **टीप:** अ‍ॅपमधील बटणे, मेनू आणि स्क्रीनवरील मजकूर प्रत्यक्षात **इंग्रजीत**च दिसतो
> (उदा. "Punch In", "Log Call"). त्यामुळे हे बटण/लेबल कुठे शोधायचे हे लगेच कळावे

> स्पष्टीकरण मराठीत दिले आहे.

## 1. हे अ‍ॅप काय करते

Rudrayani CRM हे एका कलेक्शन एजन्सीला वित्तसंस्थांच्या (Hero FinCorp, Bajaj,
TVS Credit यांसारख्या lenders च्या) वतीने थकीत कर्जाची वसुली करण्याचे काम
व्यवस्थापित करण्यास मदत करते. एजन्सी lender चा कर्ज-डेटा (loan book) इम्पोर्ट
करते, प्रत्येक खाते एका telecaller किंवा field agent कडे सोपवते (allocate करते),
आणि पहिल्या संपर्कापासून ते खाते बंद (closed) किंवा lender कडून परत घेतले
(recalled) जाईपर्यंत — प्रत्येक कॉल, भेट, पैसे देण्याचे वचन (promise-to-pay), आणि
प्रत्यक्ष जमा झालेली रक्कम यांची नोंद ठेवते.

या प्रणालीत दोन अ‍ॅप्स आहेत:

- **वेब पोर्टल** — Agency Admin, Operations Managers, आणि Team Leaders डेटा
  इम्पोर्ट करणे, काम वाटप करणे, टीमवर लक्ष ठेवणे आणि अहवाल (reports) पाहणे
  यासाठी वापरतात.
- **मोबाइल अ‍ॅप (Android)** — Telecallers आणि Field Agents त्यांची रोजची
  कामाची यादी (worklist) पाहणे, ग्राहकांना कॉल/भेट देणे, निकाल व पैसे नोंदवणे,
  आणि (Team Leaders साठी) त्यांच्या टीमवर लक्ष ठेवणे यासाठी वापरतात.

एकाच व्यक्तीकडे एकाच वेळी एकापेक्षा जास्त capability (क्षमता/अधिकार) असू शकतात
(उदा. एखादी व्यक्ती Telecaller *आणि* Team Leader दोन्हीही असू शकते) — त्या
व्यक्तीला जे करण्याची परवानगी आहे त्यानुसार अ‍ॅप आपोआप योग्य ते स्क्रीन/मेनू
दाखवते किंवा लपवते.

## 2. महत्त्वाच्या संज्ञा — शब्दकोश

| संज्ञा | अर्थ |
|---|---|
| **Agency** | तुमची कलेक्शन एजन्सी (Rudrayani Fintech). प्रणालीतील सर्व काही एकाच एजन्सीशी संबंधित असते. |
| **Company** | ज्या वित्तसंस्थेच्या/lender च्या कर्ज-वसुलीसाठी तुम्ही काम करत आहात ती संस्था (उदा. Hero FinCorp). हा तुमच्या कर्मचारी संरचनेचा भाग नाही — फक्त ग्राहक-डेटाचा स्रोत आहे. |
| **Branch** | तुमच्या एजन्सीचे एखादे प्रत्यक्ष कार्यालय/ठिकाण (उदा. सांगली, पुणे). |
| **Team** | एका Branch मधील कर्मचाऱ्यांचा गट, ज्याचे नेतृत्व एक Team Leader करतो. |
| **Capability** (क्षमता) | कर्मचाऱ्याला काय करण्याची परवानगी आहे: Agency Admin, Operations Manager, Team Leader, Telecaller, Field Agent. एका व्यक्तीकडे यापैकी अनेक असू शकतात. |
| **Customer / Loan Account** | एखाद्या company च्या फाईलमधून इम्पोर्ट केलेले, एका कर्जदाराचे थकीत कर्ज-खाते. याची स्थिती (status) असते: **Active**, **Closed**, किंवा **Recalled**. |
| **Bucket** | कर्ज किती दिवस थकीत आहे हे दर्शवणारे lender चे स्वतःचे लेबल (उदा. "30 DPD", "NPA 1"). प्रत्येक company साठी Buckets वेगळे सेट केले जातात (Buckets पान पहा) आणि वेगवेगळ्या lenders च्या लेबलांची योग्य तुलना करता यावी म्हणून एका प्रमाणित 0–20 च्या "canonical" स्केलवर mapped केले जातात. |
| **DPD** | Days Past Due — कर्जाचा EMI किती दिवस थकीत आहे. |
| **Product** | कर्जाचा प्रकार (उदा. "Personal Loan", "Home Loan"), जो इम्पोर्ट केलेल्या फाईलमधून आपोआप वाचला जातो. |
| **Allocation** | एखादे कर्ज-खाते एका विशिष्ट Telecaller किंवा Field Agent कडे सोपवणे, जेणेकरून त्यावर काम करण्याची जबाबदारी त्यांची असेल. |
| **Unallocated Queue** | इम्पोर्ट झालेली पण अजून कोणाकडेही सोपवलेली नसलेली कर्ज-खाती. |
| **Reallocation** | आधीच सोपवलेले खाते दुसऱ्या agent कडे हलवणे — यासाठी कारण (reason) देणे बंधनकारक आहे (नोंदीसाठी साठवले जाते). |
| **Reallocation Request** | एखाद्या agent ने (मोबाइल अ‍ॅपमधून) आपल्या Team Leader ला एखादा ग्राहक स्वतःकडून काढून घेण्याची विनंती करणे (उदा. चुकीचा भाग, भाषेची अडचण, वाद). Team Leader ती विनंती मंजूर (नवीन agent निवडून) किंवा नामंजूर करू शकतो. |
| **Disposition / Disposition Code** | एखाद्या कॉल किंवा भेटीचा नोंदवलेला निकाल — उदा. "PTP" (Promise to Pay), "RNR" (Ringing, No Response), "RTP" (Refuse to Pay). प्रत्येक code साठी कोणती अतिरिक्त माहिती (रक्कम, तारीख, mode, कारण, इ.) आवश्यक आहे हे आधीच ठरवलेले असते. |
| **Trail / Trail History** | एका ग्राहकावर नोंदवलेल्या प्रत्येक कॉल/भेटीचा, वेळेनुसार क्रमवार असलेला संपूर्ण इतिहास. |
| **PTP (Promise to Pay)** | ग्राहकाने विशिष्ट रक्कम विशिष्ट तारखेपर्यंत भरण्याचे दिलेले वचन. Agent ने promise-प्रकारचा disposition नोंदवला की हे आपोआप तयार होते. स्थिती: **pending** (प्रलंबित), **kept** (पाळलेले), किंवा **broken** (मोडलेले). |
| **Reminder** | Agent ने स्वतःसाठी सेट केलेली वैयक्तिक फॉलो-अप नोंद (ग्राहकासोबत किंवा त्याशिवाय) — ही PTP पेक्षा वेगळी आहे. ठरलेल्या वेळी फोनवर notification (सूचना) येते. |
| **Field Visit** | ग्राहकाच्या पत्त्यावर प्रत्यक्ष भेट दिल्याची नोंद — यासाठी फोटो आवश्यक असतो आणि GPS स्थान आपोआप नोंदवले जाते. |
| **Payment** | ग्राहकाकडून जमा झालेले पैसे — रक्कम, पद्धत (mode), तारीख, आणि (साधारणपणे) फोटो पुरावा यासह. |
| **Deposit / Deposited** | जमा केलेले पैसे प्रत्यक्ष बँकेत भरल्यानंतर, एक admin/ops वापरकर्ता ते Deposits पानावर "Deposited" म्हणून नोंदवतो — तोपर्यंत ते "Pending" (प्रलंबित) दाखवले जाते. |
| **Closed** (ग्राहक स्थिती) | खाते पूर्णपणे निकाली निघाले (पैसे भरले गेले) आणि मोबाइलवरील Payment स्क्रीनवरून बंद (closed) म्हणून चिन्हांकित केले गेले — हे सक्रिय worklist मधून निघून जाते. |
| **Recalled** (ग्राहक स्थिती) | lender ने (नवीन इम्पोर्ट फाईलद्वारे) एजन्सीला सांगितले की हे खाते यापुढे काम करण्याची गरज नाही — हे Closed पेक्षा वेगळे आहे; याचा अर्थ *lender* ने ते परत घेतले, खाते निकाली निघाले असे नाही. |
| **Normalized (या महिन्यात, lender च्या पुष्टीच्या प्रतीक्षेत)** | एखाद्या पेमेंटमुळे खात्याचा bucket lender च्या स्वतःच्या फाईलने पुष्टी करण्याआधीच "current" (नियमित) झाला असल्यास दिसणारा निळा badge — lender ची पुष्टी होईपर्यंत, इतरत्र lender चेच bucket लेबल अधिकृत मानले जाते. |
| **Bucket Movement** | ग्राहकाच्या delinquency bucket मध्ये आढळलेला बदल — एकतर "Payment (in-month)" (पेमेंटवरून लगेच ओळखलेला) किंवा "Allocation (confirmed)" (lender च्या पुढील महिन्याच्या फाईलने नंतर पुष्टी केलेला). |
| **Import Template** | एका विशिष्ट company च्या फाईल रचनेसाठी साठवलेले mapping (Excel column → system field), जेणेकरून त्याच company च्या पुढील फाईल्स पुन्हा mapping न करता आपोआप जुळतील. |
| **Import Review Queue** | पुनरावृत्त मासिक इम्पोर्टमध्ये आढळलेल्या तफावतींबाबत (discrepancies) निर्णय घेण्याची जागा — Agency Admin/Ops Manager इथे ठरवतात: नवीन कर्जे (**additions**), नवीन फाईलमधून गायब झालेली कर्जे (**removals** → recalled होतात), आणि आधी recalled झालेली कर्जे पुन्हा दिसणे (**reactivations**). कोणताही बदल आपोआप होत नाही — प्रत्येक तफावत इथे निर्णयाची वाट पाहते. |
| **Target** | प्रत्येक agent / team / branch / संपूर्ण agency साठी, पाच metrics पैकी प्रत्येकासाठी (Collection, Resolution, Roll Back, Normalization, Recovery) सेट केलेले मासिक उद्दिष्ट (₹ किंवा संख्येत). Dashboards या उद्दिष्टांविरुद्ध प्रत्यक्ष कामगिरी मोजतात. |
| **Attendance / Punch In / Punch Out** | Agent ने मोबाइल अ‍ॅपमध्ये दिवसाची कामाची पाळी (shift) सुरू/समाप्त करणे. Punch In केल्यावर स्थान-ट्रॅकिंग सुरू होते; Punch Out केल्यावर ते थांबते. |
| **Day Plan** | व्यवस्थापकांसाठी असलेले वेब पान, जे कोणत्याही दिवशी प्रत्येक agent ची उपस्थिती, प्रलंबित PTPs, प्रलंबित reminders, आणि आतापर्यंतचे काम दाखवते. |
| **Attachment / Document** | ग्राहकावर अपलोड केलेली सहाय्यक फाईल (फोटो किंवा PDF) — उदा. KYC कागदपत्र किंवा कराराची प्रत — ही payment/visit पुराव्याच्या फोटोंपेक्षा वेगळी असते. |
| **Custom Field / Detail Field** | मूळ इम्पोर्ट फाईलमधील जो column standard system field शी जुळत नाही तो "custom field" म्हणून साठवला जातो, जेणेकरून कोणताही डेटा गमावला जाणार नाही; काही fields Customer Detail view वर "detail field" म्हणून दाखवण्यासाठी चिन्हांकित करता येतात. |

## 3. भूमिका (Roles) आणि कोण काय करू शकतो

या प्रणालीत भूमिकांना **capabilities** म्हणतात, आणि त्या एकत्र (stack) होऊ
शकतात — एका व्यक्तीकडे एकाच वेळी एकापेक्षा जास्त capabilities असू शकतात.

| Capability | काय करू शकतो |
|---|---|
| **Agency Admin** (प्रत्येक agency मध्ये फक्त एकच) | सर्व काही — इतर Operations Managers जोडणे/काढणे आणि billing access यासह. Employees स्क्रीनवरून यांना edit किंवा deactivate करता येत नाही (हे वेगळ्या पद्धतीने व्यवस्थापित होते). |
| **Operations Manager** | Agency Admin करू शकतो ते सर्व — **फक्त** दुसरा Operations Manager जोडणे आणि billing सोडून. |
| **Team Leader** | ही एक *पदवी* (designation) आहे, कायमस्वरूपी पद नाही — कोणत्याही कर्मचाऱ्याला Ops Manager कडून दिली जाते. ग्राहक allocate/reallocate करू शकतो, कर्मचारी पाहू शकतो, अहवाल पाहू शकतो (स्वतःच्या टीमसाठी), tracking पाहू शकतो (स्वतःच्या टीमसाठी), आणि — मोबाइलवर — यांना अतिरिक्त "My Team" tab दिसतो. |
| **Telecaller** | फोनद्वारे नेमून दिलेल्या worklist वर काम करतो: कॉल नोंदवतो, पेमेंट्स नोंदवतो, reminders सेट करतो, reallocation विनंती करतो. फक्त स्वतःचे ग्राहक आणि स्वतःची कामगिरी पाहू शकतो. |
| **Field Agent** | Telecaller प्रमाणेच, अधिक field visits (प्रत्यक्ष भेट, फोटो + GPS सह) नोंदवू शकतो. |

उदाहरणार्थ, ज्या व्यक्तीकडे **Team Leader + Telecaller** दोन्ही capabilities
आहेत, तिला सर्वसाधारण worklist स्क्रीन्स *आणि* मोबाइलवरील "My Team" tab असे
दोन्ही मिळतात, तसेच वेब पोर्टलवर अतिरिक्त access मिळतो — एकाच भूमिकेची सक्ती
करण्याऐवजी, एखाद्या व्यक्तीकडे असलेल्या सर्व capabilities एकत्र काम करतात.

संपूर्ण अ‍ॅपमध्ये वापरली जाणारी **दृश्यमानता व्याप्ती (visibility scoping)**:
- **Agency Admin / Operations Manager** → संपूर्ण agency मधील सर्व काही पाहू शकतात.
- **Team Leader** → फक्त स्वतःची टीम पाहू शकतो (ग्राहक, अहवाल, tracking, कर्मचारी).
- **Telecaller / Field Agent** → फक्त स्वतःला सोपवलेले ग्राहक आणि स्वतःची
  कामगिरी पाहू शकतात (self-scoped).

## 4. सुरुवात कशी करावी

### वेब पोर्टल
1. तुमच्या admin ने दिलेल्या वेब पोर्टल URL वर जा.
2. तुमचा **Phone** (लॉगिन ID म्हणून वापरला जातो) आणि **Password** टाका.
3. **Log in** क्लिक करा. डावीकडील मेनू तुम्हाला ज्या पानांची परवानगी आहे तीच
   आपोआप दाखवतो.
4. पासवर्ड विसरलात? **Forgot password?** क्लिक करा — तुमचा फोन नंबर टाका,
   तुम्हाला SMS द्वारे 6-अंकी OTP मिळेल, नंतर नवीन पासवर्ड सेट करा.

### मोबाइल अ‍ॅप
1. अ‍ॅप इन्स्टॉल करा (admin कडून APK मिळेल, किंवा सेट केले असल्यास Play Store
   internal testing track वरून).
2. तुमचा **Phone Number** (10 अंक) आणि **Password** टाका, नंतर **Sign In**
   करा.
3. "Cannot reach server" असा संदेश आल्यास आधी तुमचे इंटरनेट कनेक्शन तपासा.
   तरीही समस्या राहिल्यास, admin शी संपर्क साधा — अ‍ॅप योग्य सर्व्हर पत्त्याकडे
   निर्देशित करणे आवश्यक असू शकते (login स्क्रीनच्या वर-उजवीकडील gear
   चिन्हामागील एक तांत्रिक सेटिंग; सामान्य वापरकर्त्यांनी हे बदलायची गरज नसते).
4. पहिल्या लॉगिन नंतर तुम्ही **My Worklist** वर याल. तुमच्याकडे Team Leader
   capability असल्यास, एक अतिरिक्त **My Team** tab दिसेल.

### खाते सेटअप (हे Admin/Ops Manager करतात, तुम्ही नाही)

  जाते — स्वतः नोंदणी (self-registration) करण्याची सुविधा नाही.
- इतर सर्व कर्मचारी वेब पोर्टलच्या **Employees** पानावरून जोडले जातात, जिथे
  त्यांचा फोन नंबर, सुरुवातीचा पासवर्ड, branch/team, आणि capabilities सेट केल्या
  जातात.

## 5. वेब पोर्टल — पानोपानी माहिती

डावीकडील मेनूमध्ये फक्त तुम्हाला ज्या पानांची परवानगी आहे तीच दिसतात. खाली
प्रत्येक पानासाठी कोणती permission ते उघडते हे नमूद केले आहे.

### Dashboard *(सर्वांसाठी — विशेष permission लागत नाही)*
लॉगिन नंतरचे तुमचे मुख्य पान. तुम्ही काय पाहता हे तुमच्या भूमिकेवर अवलंबून असते:
- **Agency Admin / Ops Manager / Team Leader** (`reports.view` असलेले
  कोणीही): एक संपूर्ण **Performance Dashboard** — product tabs, महिना
  निवडक (month picker), filters (तुमच्या व्याप्तीनुसार Company/Branch/Team/
  Agent/Bucket/Status), Amount/Count toggle, आणि:
  - Collection MTD (महिन्यापर्यंतचे collection) vs. target पट्टी, "महिन्यात किती
    दिवस उरले" यासह.
  - **Resolution**, **Roll Back**, **Normalization**, आणि **Recovery**
    साठी गोलाकार गेज आणि metric cards.
  - Deposited Metrics (जमा झालेले vs. deposit केलेले vs. प्रलंबित).
  - Trail Uploaded Metrics (किती काम झाले आहे याचे प्रमाण).
  - **Recalled This Month** टाईल (क्लिक केल्यास कोणते ग्राहक व का ते दिसते).
  - **Bucket Movements This Month** — पेमेंटवरून ओळखलेले vs. lender कडून
    पुष्टी झालेले.
  - **Bucket Mismatches (DPD Cross-Check)** — EMI च्या देय तारखेनुसार वेगळा
    bucket सुचवणारी "पुन्हा तपासण्यासारखी" यादी; फक्त माहितीसाठी, lender च्या
    डेटावर कधीही मात करत नाही.
  - Company/Product/Bucket/Branch/Team/Agent नुसार विभागता येणारा
    **Breakdown** तक्ता.
  - **Trail Analytics** — निवडलेल्या कालावधीतील कॉल निकाल, PTP conversion %.
  - सर्व डेटा Excel फाईल म्हणून डाउनलोड करणारे **Export** बटण.
- **Telecaller / Field Agent** (`reports.view` नसलेले): एक सोपे
  **"My Performance — &lt;तुमचे नाव&gt;"** दृश्य — फक्त तुमचे स्वतःचे आकडे,
  कोणतेही filters नाहीत. (हेच दृश्य मोबाइल अ‍ॅपवरही उपलब्ध आहे.)

### Employees *(`employees.view`)*
तुमची कर्मचारी यादी. नाव/फोनद्वारे शोधा. नवीन कर्मचारी जोडा (नाव, फोन, ईमेल,
सुरुवातीचा पासवर्ड, branch, team, आणि capability checkboxes), विद्यमान
कर्मचारी edit करा, deactivate करा, किंवा पासवर्ड रीसेट करा. फक्त ज्यांना
Operations Managers जोडण्याची परवानगी आहे तेच ती विशिष्ट capability देऊ
शकतात. Agency Admin चे खाते इथून edit करता येत नाही.

### Branches *(`branches.manage`)*
तुमच्या एजन्सीची प्रत्यक्ष कार्यालये तयार करा किंवा त्यांचे नाव बदला. डिलीट
करण्याची सुविधा नाही — branches फक्त जोडता किंवा नाव बदलता येतात.

### Teams *(`teams.manage`)*
Teams तयार करा किंवा नाव बदला; प्रत्येक team एकाच branch शी संबंधित असते.

### Companies *(`companies.manage`)*
ज्यांच्या कर्ज-वसुलीसाठी तुम्ही काम करता त्या वित्तसंस्था/lenders तयार करा किंवा
त्यांचे नाव बदला. या केवळ डेटा-स्रोत आहेत, तुमच्या संस्थेच्या रचनेचा भाग नाहीत.

### Buckets *(`companies.manage`)*
एक company निवडा, नंतर त्या company चे delinquency buckets कॉन्फिगर करा:
त्यांची क्रम (least ते most overdue) बदला, एक bucket "Current" (पूर्णपणे
नियमित — Normalization metric काम करण्यासाठी आवश्यक) म्हणून चिन्हांकित करा,
प्रत्येक bucket ला Normal किंवा NPA श्रेणी द्या (Recovery metric यावर
अवलंबून असते), आणि वेगवेगळ्या lenders ची तुलना एका स्केलवर करता यावी म्हणून
प्रत्येकाला 0–20 च्या canonical DPD क्रमांकाशी जोडा. Bucket चे *लेबल* स्वतः
इम्पोर्टमधून आपोआप येतात — ते तुम्ही इथे टाईप करत नाही, फक्त त्यांचे वर्तन
कॉन्फिगर करता.

### Import *(`imports.manage`)*
lender ची Excel फाईल प्रणालीत आणा. 4-टप्प्यांचा wizard:
1. **Select & Upload** — Company निवडा, **New customers** (हे कर्ज-पुस्तक
   पहिल्यांदाच इम्पोर्ट करत असाल तर) किंवा **Monthly allocation** (विद्यमान
   कर्जांच्या रकमा/buckets रिफ्रेश करण्यासाठी — महिना निवडा) निवडा, `.xlsx`
   फाईल अपलोड करा (कमाल 15 MB).
2. **Map Columns** — प्रत्येक Excel column ला system field शी जुळवा (Loan
   Number, Customer Name, Mobile, Product, Bucket, Due Amount, EMI, EMI Due
   Date, Agent Phone). हे mapping पुन्हा-वापरता येणारे **Template** म्हणून
   साठवा जेणेकरून याच company च्या पुढील फाईल्स आपोआप जुळतील. जो column
   जुळत नाही तो आपोआप **custom field** बनतो — कोणताही डेटा गमावला जात नाही.
3. **Preview & Validate** — आकडे पहा (वैध rows, चुका, duplicates, नवीन vs.
   विद्यमान कर्जे), row-निहाय चुकांची यादी, आणि — पुनरावृत्त मासिक
   इम्पोर्टसाठी — reactivations आढळल्यास त्याचा इशारा.
4. **Done** — इन्सर्ट/अपडेट/स्किप झालेल्यांची संख्या, आणि काही निर्णय
   घ्यायचा असल्यास **Import Review** पानाची लिंक.

प्रत्येक company साठी **Import History** tab देखील आहे. महत्त्वाचे: आधीच
लोड झालेल्या महिन्यासाठी **पुनरावृत्त** इम्पोर्ट केल्यास बदल आपोआप लागू
होत **नाहीत** — नवीन कर्जे, गायब झालेली कर्जे, आणि reactivations हे सर्व
प्रत्येक वेळी मानवी निर्णयासाठी Import Review कडे पाठवले जातात.

### Import Review *(`imports.review`)*
इम्पोर्टमधील तफावतींसाठी मंजुरी रांग. Company/Status/Type नुसार फिल्टर करा,
rows एकेक किंवा एकत्र निवडा, आणि प्रत्येकाला **Approve** किंवा **Reject** करा:
- **Addition** (निळा tag) → मंजूर केल्यास नवीन ग्राहक जोडला जातो.
- **Removal** (लाल tag) → मंजूर केल्यास ग्राहक **Recalled** म्हणून चिन्हांकित
  होतो.
- **Reactivation** (केशरी tag) → मंजूर केल्यास recalled ग्राहक पुन्हा
  Active होतो.
निर्णय घेण्यापूर्वी कोणतीही row उघडून ग्राहकाचा शेवटचा remark, प्रलंबित PTP,
आणि या महिन्यात भरलेली रक्कम पाहता येते.

### Customers *(`customers.view`)*
संपूर्ण कर्ज-पुस्तक शोधा व पहा (Company/Product/Bucket/Status नुसार फिल्टर
करा, किंवा नाव/loan number/mobile ने शोधा). कोणत्याही row वर क्लिक करून
संपूर्ण **Customer 360 view** (साईड पॅनेल) उघडा, ज्यात दिसते:
- ओळख, देय रक्कम, EMI, DPD.
- इम्पोर्टच्या वेळी नोंदवलेली अतिरिक्त "detail fields".
- **Trail History** — जुन्यापासून नव्यापर्यंत नोंदवलेला प्रत्येक कॉल.
- **Promises to Pay** — प्रत्येकाची स्थिती.
- **Payments** — रक्कम, mode, तारीख, deposited स्थिती.
- **Bucket Movements** — delinquency bucket बदलांचा इतिहास.
- **Allocation History** — प्रत्येक पुनर्वाटप (reassignment), कारणासह.
- **Documents** — सहाय्यक फाईल्स (फोटो किंवा PDF, कमाल 10MB) अपलोड/
  डाउनलोड करा.
- **Month Snapshots** — प्रत्येक मासिक इम्पोर्टच्या वेळी ग्राहकाची स्थिती.

### Allocation *(`customers.allocate`)*
दोन tabs:
- **Unallocated Queue** — फिल्टर करा, अनेक कर्जे निवडा, एक agent निवडा,
  **Assign** क्लिक करा.
- **Allocated** — कोणाकडे काय सोपवले आहे ते पहा; अनेक निवडून
  **Reallocate…** करा (कारण देणे बंधनकारक आहे, ते नोंदवले जाते); कोणत्याही
  row वर **History** क्लिक केल्यास संपूर्ण पुनर्वाटप timeline दिसते.

### Reallocation Requests *(`customers.allocate`)*
Agents नी मोबाइल अ‍ॅपवरून एखादा ग्राहक स्वतःकडून काढून घेण्यासाठी केलेल्या
विनंत्या मंजूर किंवा नामंजूर करा. **Approve** केल्यास (ऐच्छिकपणे) नवीन agent
निवडता येतो (रिकामे ठेवल्यास ग्राहक unallocated queue मध्ये परत जातो);
**Reject** केल्यास ग्राहक सध्याच्याच agent कडे राहतो. निर्णय होईपर्यंत
काहीही बदलत नाही.

### Dispositions *(`dispositions.manage`)*
Agent कॉल नोंदवताना निवडत असलेल्या call-outcome codes ची मुख्य यादी.
Code जोडा/edit करा: Action Code, Result Code, Category, Description, एक
**Remark template** (agent ने भरलेल्या माहितीवरून remark आपोआप तयार करतो),
आणि कोणती fields आवश्यक आहेत (Amount, Date, Time, Mode, Reason,
Name/Relation) यासाठी checkboxes. Codes कधीही डिलीट होत नाहीत, फक्त
**Retire** (निष्क्रिय) केले जातात (आणि पुन्हा restore करता येतात).

### Tracking *(`tracking.view`)*
दोन tabs:
- **Live Map** — प्रत्येक duty वर असलेला agent नकाशावर रंगीत ठिपका म्हणून
  दिसतो (हिरवा Moving, लाल Stationary, केशरी No Signal, राखाडी Awaiting
  First Ping), कोणी flag झाले असल्यास इशारा पट्टी; दर 30 सेकंदांनी आपोआप
  रिफ्रेश होते.
- **Route Replay** — एक कर्मचारी आणि तारीख निवडा (मागील 60 दिवसांपर्यंत —
  इतकाच काळ स्थान-डेटा साठवला जातो) आणि त्यांचा संपूर्ण दिवसाचा मार्ग
  नकाशावर, एकूण अंतरासह पहा.

हेडरमधील बेल (bell) चिन्ह (हीच permission असलेल्यांना दिसते) अ‍ॅपमध्ये कुठेही
असताना हाच live डेटा तपासत राहते आणि कोणी stationary झाले किंवा signal
गमावले तर toast सूचना दाखवते.

### Day Plan *(`tracking.view`)*
कोणत्याही तारखेसाठी, प्रत्येक agent साठी एक row: उपस्थिती स्थिती, प्रलंबित
PTPs (संख्या + ₹), प्रलंबित reminders (संख्या), केलेले कॉल्स, जमा झालेले
पेमेंट्स. या आकड्यांमागील प्रत्यक्ष ग्राहक यादी पाहण्यासाठी row expand करा.

### Targets *(`targets.manage`)*
मासिक उद्दिष्टे सेट करा. व्याप्ती स्तर निवडा (Per agent / Per team / Per
branch / Whole agency), तक्त्यातील आकडे थेट edit करा (Collection,
Resolution, Roll Back, Normalization, Recovery — ₹ किंवा संख्येत), किंवा
Excel मधून bulk-import करा. काहीतरी edit केल्यावरच **Save changes** सक्रिय
होते; एखादा cell रिकामा केल्यास ते target पूर्णपणे काढून टाकले जाते.

### Deposits *(`payments.deposit`)*
फील्डमध्ये जमा झालेली रोख रक्कम जुळवा (reconcile). महिना/status/company
नुसार फिल्टर करा, अनेक **Pending** पेमेंट्स निवडा, **Mark deposited** क्लिक
करा. आधीच deposited झालेल्या rows इथून पुन्हा uncheck करता येत नाहीत. हे
थेट Dashboard च्या Deposited Metrics ला जोडलेले आहे — वेळेवर reconciliation
केल्याने अहवाल अचूक राहतात.

खाते, आणि सर्व्हर-कनेक्शन समस्या यातील फरक स्पष्ट संदेशांद्वारे दाखवला जातो.
Gear चिन्ह (वर-उजवीकडे) हे एक तांत्रिक "server address" सेटिंग आहे —
admin ने सांगितल्याशिवाय ते बदलू नका.

### Home — खालील Tabs

- **My Team** — फक्त Team Leader capability असलेल्यांसाठी.
- **My Performance** — सर्वांसाठी.

### My Worklist
तुमचा रोजचा मुख्य तळ:
- वर **Duty banner** — **Punch In** (तुमची पाळी आणि स्थान-ट्रॅकिंग सुरू
  करते) / **Punch Out** (ते थांबवते). ट्रॅक होण्यासाठी Punch In करणे
  आवश्यक आहे; Punch In/Out साठी इंटरनेट आवश्यक आहे.
- **Sync banner** — तुम्ही ऑफलाईन असताना केलेली एखादी क्रिया अजून सर्व्हरपर्यंत
  पोहोचलेली नसेल तरच दिसते; **Sync now** ने लगेच पुन्हा प्रयत्न करता येतो.
- **Due Today strip** — आज प्रलंबित असलेले तुमचे reminders (पूर्ण झाल्यास
  checkmark वर टॅप करा) आणि प्रलंबित/overdue PTPs (फक्त पाहण्यासाठी — नवीन
  कॉल नोंदवूनच हे सोडवले जातात). Collapsible (लहान-मोठे करता येते).
- **Search** — नाव, loan number, किंवा mobile ने फिल्टर करा.
- **Customer cards** — नाव, loan number, company, देय रक्कम, शेवटच्या
  कॉलचा निकाल, असल्यास सक्रिय PTP, आणि संबंधित असल्यास "Normalized,
  lender च्या पुष्टीच्या प्रतीक्षेत" ही नोंद. ग्राहक उघडण्यासाठी card वर टॅप करा.

### Customer Detail
एका खात्याबद्दल सर्व काही:
- **Call** — थेट ग्राहकाला फोन लावते.
- **Log Call** — कॉलचा निकाल नोंदवा (खाली Call Log Screen पहा).

- **Navigate** — पत्ता नोंदवलेला असल्यास तुमचे maps अ‍ॅप ग्राहकाच्या पत्त्यावर
  उघडते.
- **Set Reminder** — या ग्राहकासाठी वैयक्तिक फॉलो-अप सूचना सेट करा.
- **⋮ मेनू → Request Reallocation** — हा ग्राहक स्वतःकडून काढून घेण्याची
  विनंती Team Leader ला पाठवा, कारणासह (आवश्यक).
- बटणांच्या खाली दिसणारी cards: **Loan Details**, **Last Disposition**,

  **Documents** (सहाय्यक फाईल्स अपलोड/पहा), आणि प्रत्येक कॉल/payment/visit/
  PTP/document एकत्र दाखवणारी **History** timeline.

### View PTPs

## 7. सुरुवातीपासून शेवटपर्यंत कार्यप्रवाह (Workflows)

### अ. नवीन कर्ज-पुस्तक (loan book) आणणे
1. **Companies** पान → lender अजून नसल्यास जोडा.
2. **Import** पान → फाईल **New customers** म्हणून अपलोड करा, columns
   जुळवा (पुढच्या वेळेसाठी Template म्हणून साठवा), preview करा, commit करा.
3. नवीन कर्जे **Unallocated Queue** मध्ये येतात.
4. Product आणि Bucket लेबल्स डेटावरून आपोआप वाचले जातात — त्यांची क्रम/
   श्रेणी/canonical mapping कॉन्फिगर करण्यासाठी **Buckets** पान तपासा.

### ब. विद्यमान पुस्तकाचे मासिक अद्ययावतीकरण
1. **Import** पान → **Monthly allocation** म्हणून अपलोड करा, महिना निवडा.
2. नवीन कर्जे, किंवा गायब झालेली कर्जे, किंवा आधी recalled झालेली कर्जे
   पुन्हा दिसणे — हे सर्व flag केले जाते.
3. **Import Review** → एक Admin/Ops Manager प्रत्येक तफावत मंजूर किंवा
   नामंजूर करतो. हे पाऊल पूर्ण होईपर्यंत काहीही बदलत नाही.
4. प्रत्येक कर्जासाठी दोन्ही प्रकारे एक **Month Snapshot** तयार होतो, जो
   ऐतिहासिक अहवालांना पुरवतो.

### क. काम वाटप करणे (Assigning work)
1. **Allocation** पान (वेब) → Unallocated Queue → rows निवडा → एक agent
   निवडा → **Assign**.
2. आता agent ला हे ग्राहक मोबाइलवरील **My Worklist** मध्ये दिसतात.
3. नंतर एखादे खाते हलवायचे आहे? त्याच पानावर, **Allocated** tab →
   **Reallocate…** (कारण आवश्यक), किंवा agent स्वतः त्यांच्या फोनवरून
   **Request Reallocation** करू शकतो, जे नंतर Team Leader मंजूर/नामंजूर
   करतो.

### ड. ग्राहकावर काम करणे (agent चा रोजचा दिनक्रम)
1. दिवसाच्या सुरुवातीला **Punch In** करा.
2. **My Worklist** मधून एक ग्राहक उघडा.
3. त्यांना **Call** करा, नंतर निकालासह **Log Call** करा — promise-प्रकारचा
   disposition आपोआप एक **PTP** तयार करतो.
4. पैसे मिळाल्यास: **Record Payment** करा (फोटो पुराव्यासह), आणि खाते
   पूर्णपणे निकाली निघाल्यास **Mark customer as Closed** toggle करा.
5. प्रत्यक्ष भेट देणाऱ्या field agents साठी: फोटोसह **Field Visit**.
6. PTPs पासून वेगळे, स्वतः नंतर लक्षात ठेवायचे असलेल्या कोणत्याही
   गोष्टीसाठी **Reminder** सेट करा.
7. ग्राहकाने दिलेली कोणतीही **Documents** जोडा.
8. दिवसाच्या शेवटी **Punch Out** करा.

### इ. व्यवस्थापकीय देखरेख (Manager oversight)
- **Day Plan** (वेब) — एका दृष्टीक्षेपात प्रत्येक agent ची उपस्थिती आणि आज
  काय प्रलंबित आहे ते तपासा, गरजेनुसार तपशीलात जा.
- **Tracking** (वेब) — agents ना नकाशावर live पहा; कोणी खूप वेळ एका
  जागी असल्यास किंवा संपर्काबाहेर गेल्यास सूचना मिळवा.
- **My Team** (मोबाइल, Team Leaders) — तीच कल्पना, प्रवासात असतानाही.
- **Dashboard** (वेब) — वैयक्तिक agent पासून संपूर्ण agency पर्यंत,
  प्रत्येक स्तरावर target विरुद्ध मासिक कामगिरी.
- **Deposits** (वेब) — जमा झालेली रोख रक्कम प्रत्यक्ष बँकेत भरलेल्या
  रकमेशी जुळवा.

### फ. Reallocation विनंतीचा जीवनक्रम
1. Agent एखाद्या ग्राहकावर (मोबाइलवर) **Request Reallocation** टॅप करतो,
   कारण देतो.
2. ही विनंती Team Leader च्या **My Team** स्क्रीनवर (मोबाइल) आणि
   **Reallocation Requests** पानावर (वेब) दिसते.
3. Team Leader **Approve** करतो (ऐच्छिकपणे नवीन agent नेमून, किंवा ग्राहक
   unallocated queue मध्ये परत पाठवून) किंवा **Reject** करतो.
4. हा निर्णय होईपर्यंत ग्राहकासाठी काहीही बदलत नाही.

## 8. Agents साठी उत्तम कार्यपद्धती (Best Practices)

खालील मार्गदर्शक तत्त्वे अ‍ॅपची वैशिष्ट्ये नाहीत — वरील साधनांचा रोजच्या
कामात प्रभावीपणे कसा वापर करावा हे सांगणारी ती कामाची पद्धत आहेत.

**Bucket नुसार कॉलचे प्रमाण.** एक ढोबळ मार्गदर्शक तत्त्व: Current
खात्यांसाठी — येणाऱ्या EMI ची आठवण करून देण्यासाठी महिन्यातून एकदा संपर्क.
सुरुवातीचे buckets (30–60 DPD) — आठवड्यातून एकदा. नंतरचे buckets (60–90
DPD आणि NPA) — आठवड्यातून एकदा किंवा अधिक, कारण निकड जास्त असते. ग्राहक
प्रतिसाद देत असेल, तर संधी असतानाच अधिक वेळा संपर्क करा. 3–5 प्रयत्नांनंतरही
ग्राहक प्रतिसाद देत नसेल, तर सतत कॉल करत राहण्याऐवजी तुमच्या Team Leader शी
याबाबत बोला.

**ग्राहक म्हणतो "पैसे भरू शकत नाही" तेव्हा.** वाद घालू नका — त्याऐवजी एखादी
लहान वचनबद्धता शोधा: "पूर्ण EMI ऐवजी काही भाग भरू शकाल का?" किंवा "तुमच्याकडे
पैसे असतील अशी एखादी विशिष्ट तारीख सांगू शकाल का?" जे ठरेल ते disposition
म्हणून नोंदवा (योग्य असल्यास promise-प्रकारच्या code सह PTP सुद्धा), आणि
ग्राहक खरोखर अडचणीत असल्याचे सांगत असल्यास ते तुमच्या Team Leader पर्यंत
पोहोचवा.

**ग्राहक म्हणतो "मी आधीच पैसे भरले आहेत" तेव्हा.** कॉलवर वाद घालू नका — आधी
ग्राहकाचा **Payments** इतिहास तपासा (Customer Detail / History विभागात
दिसतो). तिथे नोंद असल्यास, ते मान्य करून पुढे जा. नसल्यास, तुम्ही तपासून
परत कॉल कराल असे ग्राहकाला सांगा — न तपासताच तपासल्याचे नाटक करू नका —
आणि नंतर ग्राहकाने सांगितलेली रक्कम व तारीख याबाबत तुमच्या Team Leader ला
कळवा. हे कदाचित अजून प्रक्रियेत असलेले पेमेंट असू शकते किंवा खरी तफावत
असू शकते, जी तुम्ही स्वतः एकट्याने सोडवण्याचा प्रयत्न करू नये.

**बदललेले वचन अद्ययावत करणे.** विद्यमान PTP edit किंवा delete करण्याची
सुविधा नाही — ग्राहकाने पैसे कधी भरणार याबाबत विचार बदलल्यास, अद्ययावत
promise-प्रकारच्या disposition सह नवीन कॉल नोंदवा. प्रणाली एका PTP वर
दुसरे लिहिण्याऐवजी प्रत्येक PTP चा स्वतःचा इतिहास ठेवते, त्यामुळे केव्हा
काय वचन दिले गेले याची नोंद कायम राहते.

**ग्राहक वारंवार वचन मोडत असेल तर.** एकाच ग्राहकाकडून सतत (उदा. तीन किंवा
अधिक वेळा) PTP मोडली जात असल्यास ते तुमच्या Team Leader ला सांगण्यासारखे
आहे — तुम्ही सतत न पाळली जाणारी वचने घेत राहण्याऐवजी, ते खाते दुसऱ्या
agent कडे वर्ग करणे किंवा पुढे escalate करणे ते ठरवू शकतात.

**"Recalled" ग्राहकावर काम थांबवा.** ग्राहकाची स्थिती Recalled मध्ये
बदलल्यास, याचा अर्थ lender ने ते खाते परत घेतले आहे — तुमच्या वैयक्तिक
फॉलो-अप सवयींमधून ते वगळा (ते तुमच्या सक्रिय worklist मधूनही आपोआप
गायब होईल). lender ने ते भविष्यातील एखाद्या इम्पोर्टमध्ये पुन्हा पाठवले
(reactivation) तर ते पुन्हा एका agent कडे सोपवले जाईल आणि तुमच्याकडे
सोपवले गेल्यास तुम्हाला ते पुन्हा दिसेल.

**Bucket कधीही स्वतः "दुरुस्त" करण्याचा प्रयत्न करू नका.** कर्जाचा bucket
नेहमी lender च्या स्वतःच्या फाईलमधून येतो — तो अ‍ॅपमध्ये स्वहस्ते बदलण्याची
सुविधा नाही, आणि असूही नये. एखादा bucket चुकीचा वाटल्यास, तुमची शंका
कॉलच्या remark मध्ये नोंदवा आणि तुमच्या Team Leader ला सांगा; ते
Operations कडे escalate करू शकतात, जे lender कडे त्यांच्या पुढील फाईलमध्ये
सुधारणा करण्याची विनंती करू शकतात.

**तुमचे स्वतःचे target ट्रॅक करणे.** तुमची **My Performance** स्क्रीन या
महिन्यासाठीचे तुमचे target आणि आतापर्यंतचे collection दाखवते. तुम्ही
वेळापत्रकानुसार आहात का हे तपासण्याचा एक सोपा मार्ग:
`उरलेले target ÷ महिन्यात उरलेले दिवस` यावरून तुम्हाला दररोज साधारण किती
रक्कम गोळा करायची आहे हे कळते — Dashboard/Performance स्क्रीन्स आधीच तुमच्यासाठी
हाच "required per day" आकडा दाखवतात.

**तुमच्या Collection आकड्यात प्रत्यक्षात काय मोजले जाते.** फक्त प्रत्यक्ष
मिळालेले पैसेच मोजले जातात — अजून न पाळलेली PTP मोजली जात नाही. एखादी लहान
पण वास्तववादी रक्कम असलेली आणि पाळलेली PTP, नंतर मोडलेली म्हणून नोंदवली
जाणाऱ्या अवास्तव वचनापेक्षा तुमच्या आकड्यांसाठी (आणि खात्याच्या इतिहासासाठी)
जास्त मोलाची असते.

## 9. अडचणी व वारंवार विचारले जाणारे प्रश्न

**मोबाइल लॉगिनवर "Cannot reach server" दिसते.**
आधी तुमच्या फोनचे इंटरनेट कनेक्शन तपासा. कनेक्शन व्यवस्थित असूनही हे वारंवार
होत असल्यास, अ‍ॅप चुकीच्या सर्व्हर पत्त्याकडे निर्देशित असू शकते — admin शी
संपर्क साधा; हे login स्क्रीनवरील gear चिन्हाद्वारे सेट केले जाते.

**मी आधी लॉगिन केले होते पण अ‍ॅप पुन्हा सुरू केल्यावर लॉगआउट झालो.**
सामान्यपणे असे होऊ नये — इंटरनेट नसतानाही अ‍ॅप पुन्हा सुरू केल्यावर तुम्ही
लॉगिन राहता, फक्त प्रत्यक्ष authentication अयशस्वी झाल्यासच (उदा. तुमचा
पासवर्ड रीसेट झाला, किंवा तुमचे खाते deactivate झाले) लॉगआउट होते. असे
वारंवार होत असल्यास admin शी संपर्क साधा.

**खाते Locked (लॉक) झाले.**
सलग अनेक वेळा चुकीचा पासवर्ड टाकल्यास खाते तात्पुरते लॉक होते. तुमच्या
manager शी संपर्क साधा (फक्त Admin/Ops Manager पासवर्ड रीसेट करू शकतो).

**मी अपलोड केलेला फोटो/document "will sync automatically" असे दाखवतो —
तो साठवला गेला आहे का?**
हो. तो तुमच्या फोनवर सुरक्षित साठवला गेला आहे आणि पुन्हा कनेक्शन मिळताच
(किंवा worklist स्क्रीनवरील **Sync now** टॅप केल्यावर) अपलोड होईल. अ‍ॅप बंद
केले तरी तो नंतर sync होईल.

**ऑफलाईन असताना PDF अपलोड अयशस्वी झाले.**
फोटोंप्रमाणे नाही — PDF documents अपलोड करण्यासाठी सक्रिय इंटरनेट कनेक्शन
आवश्यक आहे, ते रांगेत साठत नाहीत. पुन्हा ऑनलाईन आल्यावर प्रयत्न करा.

**मला "My Team" tab का दिसत नाही?**
फक्त Team Leader capability असलेल्या खात्यांनाच हे दिसते. तुम्हाला ही
capability मिळायला हवी असल्यास, Employees पानावरून ती देण्यासाठी तुमच्या
Operations Manager शी संपर्क साधा.

**मी ज्या ग्राहकावर काम करत आहे तो "Normalized this month, pending lender
confirmation" असे का दाखवतो?**
एका पेमेंटमुळे खाते पुन्हा नियमित (current) झाले आहे, पण lender ने अजून
त्यांच्या स्वतःच्या फाईलमध्ये त्याची पुष्टी केलेली नाही. यामुळे तुम्ही काय
करावे यात काहीही बदल होत नाही — नेहमीप्रमाणे खात्यावर काम सुरू ठेवा.

**Closed आणि Recalled यात काय फरक आहे?**
**Closed** म्हणजे खाते पूर्णपणे निकाली निघाले (पैसे भरले गेले) आणि अ‍ॅपवरून
बंद म्हणून चिन्हांकित केले गेले. **Recalled** म्हणजे *lender* ने ते खाते
परत घेतले (साधारणपणे मासिक इम्पोर्टमध्ये दिसते) — हे agent करत नाही, आणि
याचा अर्थ कर्ज निकाली निघाले असा होत नाही.

**मी reminder सेट केला पण notification मिळाले नाही.**
तुमच्या फोनच्या सेटिंग्जमध्ये या अ‍ॅपसाठी notifications ला परवानगी आहे
याची खात्री करा (पहिल्यांदा reminder सेट करताना यासाठी विचारले जायला हवे).
तुम्ही ऑनलाईन असताना अ‍ॅप उघडताच reminders आपोआप पुन्हा शेड्युल होतात,
त्यामुळे अ‍ॅप किंवा फोन पुन्हा सुरू केल्याने ते गमावले जाऊ नयेत.

**माझी स्वतःची कामगिरी मला कुठे दिसेल?**
मोबाइलवरील **My Performance** tab, किंवा वेबवरील Dashboard पान (जर
तुमच्याकडे manager-स्तरावरील report access नसेल तर वैयक्तिक स्कोअरकार्ड
दाखवते).
