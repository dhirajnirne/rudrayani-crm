3 FIRST IMPORT Import Month 1 Allocation

http://localhost:5173/imports

Steps:

  1. Click "Upload File" or drag file
  2. File: backend/test/fixtures/alpha-finance-month1.xlsx
  3. Column mapping auto-fills (loan_agreement_no → loan_number, etc.)
  4. Select detail fields: branch, state, zone (optional)
  5. Click "Save as Template" → name it "Hero Finance - Standard"
  6. Preview shows: "8 rows will be inserted"
  7. Click "Commit Import"
✓ Expected: Success message. Dashboard Allocated Count updates to 8.

4 REPEAT IMPORT Mid-Month Refresh (With Additions & Removals)

http://localhost:5173/imports

Scenario: New allocation file arrives mid-month

  1. Upload: alpha-finance-month2-refresh.xlsx
  2. Use saved template "Hero Finance - Standard"
  3. Preview shows:
    • 7 will be updated (existing loans with changed amounts)
    • 1 addition pending review (ALPHA-009)
    • 1 removal pending review (ALPHA-004 missing from file)
  4. Commit import
⚠️ Important: Additions and removals don't auto-apply. They go to the Import Review queue for approval.

5 Review Queue

http://localhost:5173/reviews

You'll see a badge showing "2 Pending"

  • Two pending items: 1 addition, 1 removal
  • Click on each to see full payload
  • For addition (ALPHA-009): Click "Approve" → customer inserted
  • For removal (ALPHA-004): Click "Approve" → customer status becomes "Recalled"
✓ After approving:
  • ALPHA-009 now appears in Allocations (Active)
  • ALPHA-004 now appears as Recalled
  • Dashboard updated accordingly

6 Configure Buckets (Canonical Mapping)

http://localhost:5173/buckets

Map DPD canonical buckets:

  • X → Canonical: 0 (current month)
  • 1 → Canonical: 1 (30-60 DPD)
  • 2 → Canonical: 2 (60-90 DPD)
  • NPA → Canonical: 3 (90+ DPD)
✓ After mapping: Dashboard "Bucket Mismatches" card now shows live data

7 View DPD Cross-Check Results

http://localhost:5173/dashboard → Scroll to "Bucket Mismatches"

You'll see:

  • ALPHA-008: Lender says "X" (canonical 0) but due_date is 75 days old
  • System computed: 75÷30 = 2.5 → canonical 2 (mismatch!)
  • Other customers showing agreement (not flagged)
💡 Insight: This report helps identify loans that might be mis-bucketed by the lender. Use it to contact them and request corrections.

8 Customer 360° View

http://localhost:5173/allocations → Click any customer row

Drawer opens with 8 sections:

  1. Identity: Loan #, Name, Company, Bucket, Due Amount, EMI
  2. Detail Fields: Branch, State, Zone (from import)
  3. Trail History: Last 50 calls with timestamps and remarks
  4. PTPs: Promised To Pay commitments (status, amount, date)
  5. Payments: All payments received
  6. Bucket Movements: Payment-driven and allocation-confirmed events
  7. Allocation History: Who's been assigned to this customer
  8. Month Snapshots: Historical state for each month
👁️ Complete picture: All customer data in one place. No need to cross-reference multiple screens.

9 Allocations & Customer Assignment

http://localhost:5173/allocations

For Team Leaders:

  • See all active customers
  • Filter: Unallocated, Active, Recalled, Closed
  • Right-click unallocated customer
  • Select "Assign to Agent" → choose agent name
  • ✓ Agent column updates, allocation log created
🎯 Balance workload: ~30-50 active cases per agent. Monitor daily to ensure even distribution.

10 Export & Reports

http://localhost:5173/dashboard → [Export to Excel]

Downloads workbook with 8 sheets:

  1. Summary (headline metrics)
  2. Agents (per-agent performance)
  3. Breakdown (by selected dimension)
  4. Trail (call activity + PTP conversion)
  5. Recalls (detailed customer list)
  6. Bucket Movements (events)
  7. Bucket Mismatches (DPD cross-check)
  8. Raw Data (all snapshots)
📊 All filters apply: Month, Company, Team, Agent, Product, Bucket, Status
